# Security Policy

## Supported Versions

Only latest commit is supported.

Older commits may contain known vulnerabilities.

---

## Reporting Vulnerabilities

DO NOT open public issue.

Send report privately with:

- description
- reproduction steps
- impact analysis
- suggested fix (optional)

---

## Response SLA

| Severity | Response |
|--------|----------|
Critical | < 24h |
High | < 48h |
Medium | < 3 days |
Low | < 7 days |

---

## Security Principles

This project enforces:

- least privilege
- network isolation
- container separation
- TLS encryption
- minimal attack surface

---

## Security Scope

Covered:

- deployment script
- container configs
- network configs

Not covered:

- host OS misconfiguration
- third party packages
- user mistakes

---

## Responsible Disclosure

Researchers will be credited after patch release.
